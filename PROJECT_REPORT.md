# تقرير شامل عن مشروع **منظومة الأكاديمي الذكي**

> هذا التقرير يصف المشروع كما هو حالياً بعد التطوير، مع توثيق الفكرة، الهيكل، التقنيات، التدفقات، ونقاط التوسع المستقبلية.

---

## 1. نظرة عامة على المشروع

- **اسم المشروع**: منظومة الأكاديمي الذكي
- **نوع المشروع**: تطبيق ويب أحادي الصفحة (Single Page Application) يعتمد على واجهة تفاعلية مع ذكاء اصطناعي.
- **الهدف الرئيسي**: مساعدة الطلاب والباحثين في العالم العربي على إعداد **بحوث وتقارير أكاديمية بالعربية** بشكل منظم واحترافي، مع إمكانية تصدير النتائج إلى ملفات Word وPDF جاهزة للطباعة.
- **المستخدمون المستهدفون**:
  - طلاب الجامعات (بكالوريوس، ماجستير، دكتوراه).
  - الباحثون والأكاديميون.
  - المعلمون والمدربون الذين يحتاجون تقارير/مواد مكتوبة.

---

## 2. التقنيات المستخدمة

- **الواجهة الأمامية (Front-end)**:
  - `React 19` مع `TypeScript`.
  - أداة البناء والتشغيل: `Vite`.
  - تصميم واجهات: `Tailwind CSS 4` + إضافة `@tailwindcss/typography`.
  - أيقونات: `lucide-react`.
  - عرض النص المنسق: `react-markdown` + `remark-gfm`.

- **الخلفية (Back-end)**:
  - `Node.js` مع `Express`.
  - إدارة المتغيرات السرية: `dotenv`.

- **الذكاء الاصطناعي (AI)**:
  - مكتبة `@google/genai`.
  - نموذج: `gemini-3.1-pro-preview`.
  - استخدام إعداد التفكير المتقدم: `ThinkingLevel.HIGH`.

- **أدوات أخرى**:
  - `tsx` لتشغيل TypeScript على الخادم.
  - إعداد `proxy` في Vite لتوجيه الطلبات إلى خادم Express.

### 3.1. ملفات الجذر

- **`index.html`**
  - صفحة HTML الأساسية.
  - اللغة والاتجاه مضبوطان على العربية: `lang="ar" dir="rtl"`.
  - تحتوي على عنصر الجذر `#root` الذي تُركّب داخله تطبيق React.

- **`package.json`**
  - تعريف اعتمادات المشروع (dependencies / devDependencies).
  - سكربتات مهمة:
    - `dev`: تشغيل واجهة Vite على المنفذ 3000.
    - `server`: تشغيل خادم Express على المنفذ 3001.
    - `dev:full`: تشغيل الخادم والواجهة معًا (يتطلب إعداد `npm-run-all` إن تم استخدامه).

- **`vite.config.ts`**
  - تهيئة Vite مع:
    - إضافة React.
    - إضافة Tailwind.
    - ضبط alias `@` للجذر.
    - إعداد `proxy` لمسار `/api` إلى `http://localhost:3001` (خادم Express).

- **`tsconfig.json`**
  - إعدادات TypeScript للمشروع (المستوى المستهدف ES2022، دعم DOM، React JSX، إلخ).

- **`.env.example`**
  - نموذج لملف البيئة، يحتوي على المتغير المهم:
    - `GEMINI_API_KEY` (يجب نسخه إلى `.env` الفعلي).

### 3.2. مجلد `src/`

- **`main.tsx`**
  - نقطة الدخول لتطبيق React.
  - يقوم برندر `App` داخل عنصر `#root` مع `StrictMode`.

- **`index.css`**
  - استيراد أساسيات Tailwind:
    - `@import "tailwindcss";`
    - `@plugin "@tailwindcss/typography";`

- **`App.tsx`**
  - مكوّن الجذر للتطبيق.
  - يدير التبديل بين:
    - `LandingPage` (صفحة الهبوط).
    - `ChatApp` (واجهة الأكاديمي الذكي).
  - المنطق بسيط: حالة `showChat` للتحكم في ما يُعرض.

#### 3.2.1. مجلد `src/components`

- **`LandingPage.tsx`**
  - صفحة هبوط عربية بالكامل تعرّف بـ **منظومة الأكاديمي الذكي**.
  - الأقسام الرئيسية:
    - شريط علوي (Navbar) يحتوي على:
      - شعار المنظومة.
      - روابط:
        - `آلية العمل` → يوجّه إلى قسم `#how-it-works`.
        - `من نحن` → يوجّه إلى قسم `#about-us`.
      - زر "ابدأ الآن" الذي يستدعي `onGetStarted` للانتقال لواجهة الدردشة.
    - قسم رئيسي (Hero):
      - عنوان قوي يشرح الفكرة: إنجاز البحوث والتقارير الجامعية بالعربية باحترافية خلال دقائق.
      - وصف موجز لدور المنظومة.
      - أزرار:
        - "ابدأ بحثاً جديداً الآن".
        - "شاهد مثال لبحث جاهز".
    - قسم إحصائيات (Stats):
      - عدد تقريبي للمستخدمين/المواضيع/المجالات (قيم تجريبية حالياً).
    - قسم **آلية العمل** `#how-it-works`:
      - ثلاث خطوات تفاعلية:
        1. تحديد موضوع البحث.
        2. توليد محتوى أكاديمي منسق.
        3. مراجعة وتصدير الملف.
    - قسم **من نحن** `#about-us`:
      - تعريف برؤية المشروع وأهدافه.
      - التأكيد على الاستخدام المسؤول للذكاء الاصطناعي (مساعدة لا استبدال).
    - تذييل (Footer):
      - روابط لسياسة الخصوصية، شروط الاستخدام، التنبيهات القانونية.
      - أيقونات اتصال/مشاركة.

- **`ChatApp.tsx`**
  - واجهة المساعد الأكاديمي الذكي.
  - الوظائف الأساسية:
    - إدارة المحادثة بين المستخدم والنموذج (`messages`).
    - إدارة الجلسات (سجل الأبحاث) عبر `sessions`.
    - التكامل مع خادم `/api/chat`.
    - التصدير إلى ملفات Word وPDF.

  - الحالة (State):
    - `messages`: قائمة الرسائل (مستخدم / نموذج).
    - `sessions`: قائمة الجلسات المخزنة.
    - `activeSessionId`: الجلسة الحالية النشطة.
    - `input`: النص الحالي في مربع الإدخال.
    - `isLoading`: حالة تحميل عند انتظار رد النموذج.
    - `isSidebarOpen`: إظهار/إخفاء الشريط الجانبي على الشاشات الصغيرة.

  - المزايا في الواجهة:
    - واجهة RTL كاملة بالعربية.
    - شاشة ترحيبية عند عدم وجود رسائل مع أمثلة جاهزة لبدء البحث.
    - عرض رسائل المستخدم والنموذج بفواصل وألوان واضحة.
    - استخدام `ReactMarkdown` + `remark-gfm` لعرض مخرجات النموذج المنسقة (Markdown).
    - أزرار أسفل كل رد من الموديل:
      - **Word**: توليد ملف `.doc` منسق بالعربية وجاهز للطباعة.
      - **PDF**: فتح نافذة الطباعة لتحويل المحتوى إلى PDF.
    - رسالة تنبيه أسفل الإدخال تحثّ على مراجعة النتائج.

  - سجل الأبحاث (Sessions):
    - عند بدء "بحث جديد" يتم إنشاء جلسة جديدة بعنوان افتراضي "بحث جديد".
    - عند إرسال أول رسالة، يتم اقتراح عنوان للجلسة من أول سطر من طلب المستخدم (قصير ومختصر).
    - الجلسات والرسائل تُخزن في `localStorage`:
      - `academic_assistant_sessions`
      - `academic_assistant_messages`
      - `academic_assistant_active_session`
      - بالإضافة إلى تخزين مخصص لكل جلسة (مثل `academic_assistant_messages_<sessionId>`).
    - الشريط الجانبي يعرض قائمة الجلسات مع:
      - العنوان.
      - تاريخ ووقت الإنشاء بالتنسيق العربي.
    - يمكن للمستخدم اختيار جلسة لاسترجاع رسائلها والعودة لعرضها.

#### 3.2.2. مجلد `src/config`

- **`aiConfig.ts`**
  - تعريف الإعدادات العامة للذكاء الاصطناعي:
    - `AI_SYSTEM_INSTRUCTION`: نص عربي يحدد شخصية المساعد (أكاديمي ذكي) ومعاييره في الإجابة.
    - `AI_MODEL_NAME`: اسم النموذج المستخدم (`gemini-3.1-pro-preview`).
  - الهدف من هذا الملف هو **مركزية الإعدادات** لتسهيل تعديلها لاحقاً (تغيير النموذج/التعليمات دون لمس المنطق).

### 3.3. مجلد `server/`

- **`server/index.ts`**
  - خادم Express بسيط يعمل كبوابة بين الواجهة الأمامية وخدمة Gemini.
  - يستخدم:
    - `dotenv` لتحميل `GEMINI_API_KEY` من `.env`.
    - `cors` و `express.json()` لتسهيل استهلاك الـ API.
    - `@google/genai` لاستدعاء نموذج Gemini.
  - مسار API:
    - `POST /api/chat`
      - **المدخلات (Body)**:
        - `{ messages: { role: 'user' | 'model', content: string }[] }`
      - **الخطوات**:
        1. التحقق من وجود المفتاح `GEMINI_API_KEY`.
        2. التحقق من صحة بنية الرسائل.
        3. تحويل الرسائل إلى الصيغة التي يقبلها نموذج Gemini (`contents`).
        4. استدعاء `generateContent` مع:
           - `model: gemini-3.1-pro-preview`.
           - `systemInstruction`: نفس موجه النظام العربي.
           - `thinkingConfig`: بمستوى تفكير مرتفع.
        5. إعادة النص الناتج كـ `{ content: string }`.
      - **معالجة الأخطاء**:
        - إعادة رسائل خطأ عربية مناسبة في حال:
          - غياب المفتاح.
          - خطأ غير متوقع في الاستدعاء.

## 4. تدفق العمل الأساسي (User Flow)

1. **زيارة صفحة الهبوط**:
   - المستخدم يفتح `http://localhost:3000`.
   - يرى صفحة هبوط عربية مع تعريف بالمنظومة وروابط "آلية العمل" و"من نحن".

2. **بدء استخدام المساعد**:
   - يضغط على زر "ابدأ الآن" أو "ابدأ بحثاً جديداً الآن".
   - يتحول التطبيق لعرض مكوّن `ChatApp`.

3. **كتابة موضوع البحث**:
   - يمكن للمستخدم:
     - كتابة موضوعه الخاص في مربع الإدخال.
     - أو اختيار أحد الأمثلة الجاهزة لملء النص تلقائياً.

4. **إرسال الطلب إلى النموذج**:
   - عند الضغط على زر الإرسال أو Enter:
     - تُضاف رسالة المستخدم إلى `messages`.
     - تُرسل جميع الرسائل (كسياق محادثة) إلى مسار `/api/chat`.

5. **توليد المحتوى في الخادم**:
   - خادم Express يستقبل الرسائل.
   - يستدعي نموذج Gemini مع تعليمات النظام العربية.
   - يسترجع النص المولد (بحث/تقرير) ويرسله إلى الفرونت.

6. **عرض الرد في الواجهة**:
   - يتم إضافة رسالة جديدة من `role: 'model'` إلى `messages`.
   - تُعرض الرسالة باستخدام `ReactMarkdown` منسّقة بالعربية (عناوين، فقرات، قوائم...).

7. **تصدير البحث**:
   - يمكن للمستخدم:
     - الضغط على "Word" لتوليد ملف `.doc` بتنسيق مناسب للطباعة.
     - الضغط على "PDF" لفتح نافذة الطباعة وحفظ الملف كـ PDF.

8. **إدارة الجلسات**:
   - الضغط على "بحث جديد" يفتح جلسة جديدة ويعيد تعيين الرسائل.
   - يمكن من الشريط الجانبي استعراض الجلسات السابقة والعودة إليها.

## 5. نقاط القوة الحالية

- واجهة عربية حديثة ومريحة للمستخدم.
- تركيز واضح على حالة استخدام واحدة مهمة: **البحوث والتقارير الأكاديمية بالعربية**.
- استخدام نموذج متقدم (Gemini) مع تعليمات نظام مصممة خصيصاً للاستخدام الأكاديمي.
- فصل مفتاح الذكاء الاصطناعي عن الواجهة بحمايته في خادم Express.
- وجود سجل جلسات بسيط يتيح للمستخدم العودة لبعض الأبحاث السابقة.
- تصدير مباشر إلى تنسيقات يحتاجها المستخدم عملياً (Word / PDF).

---

## 6. نقاط يمكن تحسينها لاحقاً

- **إدارة المستخدمين**:
  - إضافة نظام حسابات (تسجيل دخول)، بحيث تصبح الجلسات مرتبطة بالمستخدم بدلاً من الجهاز فقط.

- **تخزين احترافي للسجل**:
  - استخدام قاعدة بيانات (مثل SQLite / PostgreSQL) بدلاً من `localStorage` فقط.

- **قوالب متخصصة**:
  - قوالب جاهزة لأنواع معينة من المستندات:
    - خطة بحث.
    - مراجعة أدبيات.
    - تقرير عملي.
    - ملخص بحث.

- **تحسين تجربة التحرير**:
  - إضافة محرر نصي غني (Rich Text Editor) لدعم التعديل اليدوي بشكل أفضل قبل التصدير.

- **تكاملات خارجية**:
  - الحفظ مباشرة في Google Drive أو OneDrive.
  - إرسال المستند عبر البريد الإلكتروني من داخل المنصة.

- **اختبارات آلية**:
  - إضافة اختبارات وحدات واختبارات واجهة لضمان استقرار المنظومة مع أي تطوير جديد.

## 7. طريقة التشغيل (ملخص سريع)

1. إنشاء ملف `.env` في الجذر ونسخ محتوى `.env.example` إليه، ثم وضع قيمة صحيحة لـ `GEMINI_API_KEY`.
2. تثبيت الحزم:

   ```bash
   npm install
   ```

3. تشغيل الخادم:

   ```bash
   npm run server
   ```

4. تشغيل الواجهة:

   ```bash
   npm run dev
   ```

5. فتح المتصفح على:

   ```text
   http://localhost:3000
   ```

وبذلك تكون منظومة الأكاديمي الذكي جاهزة للاستخدام محلياً، مع واجهة عربية وتجربة متكاملة لإنتاج البحوث والتقارير الأكاديمية.
